# Tutorial / Onboarding — Design

Дата: 2026-06-17
Статус: утверждён, готов к плану реализации
Ветка: `feat/tutorial` (от `feat/juice`)

## Контекст

PancakeFlip — прототип про жарку блинов; одна сцена, UI/сцена генерируются
`PancakeFlipSetup.BuildEverything()`. Базовый цикл: выбрать заказ → открыть ингредиенты →
собрать блин → поставить готовиться → подбросить и прожарить обе стороны → сдать. Состояние
читается через уже существующий API:
`GameSession.ActiveOrder`, `GameSession.Build.Slots`, `GameSession.HasCookingPancake`,
`GameSession.IsPancakeCookedEnough()`, событие `GameSession.OnServed`. Магазин ингредиентов —
`IngredientsScreenView` (модалка). Сейвов нет, кроме того что добавим (PlayerPrefs).

Цель: онбординг, обучающий **ядру-циклу**, пошагово, с подсветкой целевого элемента и
блокировкой постороннего ввода, запуск **один раз** (помним в PlayerPrefs) с кнопкой
«Пропустить». Подход — **предикатный `TutorialController`** (читает существующее состояние,
почти без правок геймплея).

## Принципы

- Декаплинг: тутор читает состояние и существующее событие `OnServed`; не добавляем события
  по всему коду.
- Без ассетов: маска-«дырка» — четыре затемняющие панели вокруг целевого прямоугольника
  (без шейдеров). Подсказка/кнопки — TMP/uGUI кодом.
- Поверх всего: туторный Canvas с `sortingOrder` выше модалок (модалки = 100 → тутор = 200).
- Безопасно: нет цели/ссылки — шаг деградирует (показывает подсказку без выреза), без
  исключений.

## Объём (по решению)

Только ядро: заказ → открыть ингредиенты → собрать → готовить → подбросить/прожарить → сдать.

## Компоненты

### 1. `TutorialProgress` (чистое ядро, тестируемое)

Сборка `IdlePancake.PancakeFlip.TutorialCore` (без UnityEngine). Хранит индекс шага и счётчик.

- `int StepCount`, `int CurrentIndex`, `bool IsComplete`.
- `void Advance()` — переход к следующему шагу (кламп на конце → `IsComplete`).
- ctor `TutorialProgress(int stepCount)`.

Юнит-ответственность: «на каком шаге сейчас и закончили ли». Не знает про Unity/предикаты.

### 2. `TutorialStep` (данные шага, рантайм)

Простой класс (не SO): `string text`, `RectTransform target` (nullable),
`System.Func<bool> isComplete`, `bool blockInput`. Создаются кодом в `TutorialController`.

### 3. `TutorialController` (MonoBehaviour) — проигрывание

Ссылки (через инспектор, проставляются в `BuildEverything`): целевые `RectTransform`'ы
(список заказов, кнопка ингредиентов, контейнер ингредиентов, кнопка «Готовить», игровое
поле/центр, кнопка «Сдать»), ссылка на `IngredientsScreenView` (для предиката «экран
активен»), и собственный оверлей (4 панели-маски, подсказка, кнопка «Пропустить»).

Поведение:
- На `Start`: если `PlayerPrefs.GetInt(DoneKey)==1` — выключиться. Иначе построить шаги и
  начать с шага 0.
- Шаги (в порядке), `isComplete` — предикаты по существующему API:
  1. target=список заказов; text=«Выбери заказ»; `() => GameSession.Instance.ActiveOrder != null`.
  2. target=кнопка ингредиентов; «Открой ингредиенты»; `() => ingredientsScreen.isActiveAndEnabled`.
  3. target=контейнер ингредиентов; «Собери блин — добавь ингредиенты»;
     `() => GameSession.Instance.Build != null && GameSession.Instance.Build.Slots.Count > 0`.
  4. target=кнопка «Готовить»; «Поставь на сковороду»; `() => GameSession.Instance.HasCookingPancake`.
  5. target=игровое поле (центр); «Подбрось и прожарь обе стороны»; `blockInput=false`
     (ввод подброса не блокируем); `() => GameSession.Instance.IsPancakeCookedEnough()`.
  6. target=кнопка «Сдать»; «Сдай заказ»; `() => _served` (флаг от `OnServed`).
- `Update`: если текущий шаг завершён по предикату — `Advance()`, перестроить оверлей под
  новый шаг; если `IsComplete` — завершить (скрыть оверлей, `PlayerPrefs.SetInt(DoneKey,1)`).
- «Пропустить» — сразу завершить и пометить done.
- Оверлей под целевой прямоугольник: считать экранный rect target (через его углы), выставить
  4 панели (сверху/снизу/слева/справа), оставив «дырку»; панели — raycast-блок. Если
  `blockInput==false` или target==null — панели прозрачны/выключены (ввод свободен). Подсказка
  ставится рядом с дыркой.
- Подписка на `GameSession.OnServed` (флаг `_served`) в `Start`/`OnEnable`, отписка в `OnDestroy`.

### 4. PlayerPrefs

- Ключ `DoneKey = "pf_tutorial_done"`. Завершение/Пропуск → `SetInt(1)`.
- Сброс для тестов: `[MenuItem("PancakeFlip/Сбросить тутор")]` в редакторе
  (`PlayerPrefs.DeleteKey`). Чтобы можно было прогнать повторно.

### 5. Проводка в `BuildEverything`

- Создать туторный Canvas (overlay, sortingOrder 200) + `TutorialController` + оверлей-объекты
  (4 панели, подсказка-TMP, кнопка «Пропустить»).
- Проставить целевые ссылки на уже создаваемые элементы: `orderPanel`, кнопка ингредиентов
  (`recipesBtn`/shop), контейнер ингредиентов из `IngredientsScreenView`, кнопку «Готовить»
  (`iCookBtn`), центр игрового поля (`fxCenter`), кнопку «Сдать», и `IngredientsScreenView`.
  (Точные локали уточняются на этапе плана по тексту `BuildEverything`.)

## Поток данных

```
Start: PlayerPrefs done? --нет--> build steps --> show overlay(step0)
Update: step.isComplete()? --да--> Advance --> rebuild overlay / finish
OnServed --> _served=true (для шага 6)
Skip --> finish + PlayerPrefs done
```

## Краевые случаи

- Цель внутри модалки (кнопка «Готовить») — оверлей выше модалки, «дырка» считается от
  экранного rect кнопки независимо от её Canvas.
- Ресайз/смена ориентации во время тутора — оверлей пересчитывается каждый кадр от текущего
  rect цели.
- Игрок закрыл магазин на шаге 3/4 — предикат не выполнен, тутор ждёт; подсказка остаётся
  (можно вернуть инструкцию «открой ингредиенты», но для прототипа достаточно ожидания).
- Нет `GameSession.Instance` на старте — `Start` тутора после `GameSession.Awake`
  (execution order GameSession = -100); предикаты null-безопасны.

## Проверка (Unity)

- EditMode-тесты `TutorialProgress` зелёные.
- Первый запуск (после сброса тутора): идёт пошаговый онбординг, подсветка ведёт по 6 шагам,
  посторонние клики заблокированы (кроме шага подброса), «Пропустить» завершает.
- Повторный запуск — тутор не появляется (PlayerPrefs).
- Портрет и ландшафт — «дырка»/подсказка корректно вокруг целей.

## Вне scope

- Обучение магазину-апгрейдов и карте (другой цикл).
- Озвучка/анимированная рука-указатель (достаточно подсветки + текста).
- Локализация.
