# Global Map v2 — Level Gate + Purchase — Design

Дата: 2026-06-17
Статус: утверждён, готов к плану реализации
Ветка: `feat/global-map-v2` (от `feat/tutorial`)

## Контекст

Глобальная карта v1: линейная цепочка, разблокировка следующей локации по N выполненных
заказов, бесплатный «переезд», экран карты — ряд узлов. Игрок добавил арт
(`Assets/Art/PancakeFlip/GlobalMap/`: `GlobalMap.png` — фон, `City01–03.png` — иконки
городов, `Blocked.png` — замок, `Car.png`) и хочет другую модель и оформление.

Цель v2: карта-фон с точками-городами; у каждого города три состояния:
- **Owned** (куплен) → зелёная кнопка **«Войти»** (вход в город).
- **Buyable** (уровень достигнут, не куплен) → жёлтая **«Купить»** → модалка покупки.
- **Locked** (уровень < требуемого) → иконка `Blocked` + красная **«Закрыто»**; по клику —
  сообщение «Требуется уровень N».

Решения брейншторма: цепочка по N заказов **убирается**; гейт — по уровню игрока + покупка
за монеты; стартовый город (Заправка) куплен бесплатно; в любой купленный город можно
свободно «Войти»; покупки помним **только на сессию** (как монеты/уровень). Гибрид-модель
контента (рецепты/ингредиенты копятся, покупатели/спрос локальные) сохраняется.

## Принципы

- `MapState` остаётся чистым (тестируемым), без UnityEngine; уровень игрока передаётся
  параметром, монеты тратит `GameSession` (`Wallet`).
- Позиции маркеров — данные (`LocationConfig.mapPosition`, нормализованные 0..1), выставляются
  в инспекторе по фону. Число городов — любое (через `WorldMapConfig`).
- Модалка покупки — в стиле существующих модалок (панель + заголовок + цена + кнопка
  «Купить» + крестик справа-сверху).
- Без правок несвязанного. Юс-эффекты подключаются по событию покупки.

## Компоненты

### 1. `MapState` (переписать чистое ядро)

Заменяет линейную логику на владение городами. Без UnityEngine.

- ctor `MapState(int[] requiredLevels, int startOwnedIndex = 0)` — `owned[startOwnedIndex]=true`,
  `CurrentIndex = startOwnedIndex`.
- `int LocationCount`, `int CurrentIndex`.
- enum `CityState { Owned, Buyable, Locked }`.
- `CityState StateOf(int index, int playerLevel)` — Owned, иначе Locked если
  `playerLevel < requiredLevels[index]`, иначе Buyable.
- `bool CanEnter(int index)` — `owned[index]`.
- `bool CanBuy(int index, int playerLevel)` — `!owned[index] && playerLevel >= requiredLevels[index]`.
- `void MarkOwned(int index)`.
- `bool SetCurrent(int index)` — только если owned; ставит `CurrentIndex`, иначе false.
- `bool IsOwned(int index)`, `int RequiredLevel(int index)`.
- event `OnChanged`.

Тесты (EditMode): стартовый owned/остальные по уровню; `StateOf` три состояния по уровню;
`CanBuy`/`CanEnter`; `MarkOwned` переводит в Owned и разрешает Enter; `SetCurrent` только
для owned; границы индексов.

### 2. `LocationConfig` (добавить поля)

К существующему добавить: `int requiredLevel = 1;`, `int cityCost = 0;`,
`Vector2 mapPosition = new(0.5f, 0.5f);`. `mapIcon` (уже есть) — спрайт города (City01/02/03).

### 3. `GameSession` (правки)

- `Map = new MapState(requiredLevels, 0)` (город 0 куплен). `requiredLevels` из `worldMap`.
- `bool TryBuyCity(int index)`: `Map.CanBuy(index, Wallet.Level)` и
  `Wallet.SpendCoins(worldMap.locations[index].cityCost)` → `Map.MarkOwned(index)` →
  `OnNewLocationUnlocked?.Invoke()` (юс-эффект). Возвращает успех.
- `TravelTo(int index)` (вход в город): теперь требует `Map.CanEnter(index)` (куплен), иначе
  no-op; далее применяет контент локации (рецепты/ингредиенты — один раз на город, через
  `HashSet<int> _contentApplied` в самом `GameSession`; спрос/покупатели — каждый вход) и
  `Map.SetCurrent(index)`.
- Убрать вызов `Map.RecordOrderCompleted()` из `TryServe` (механики заказов-для-разблокировки
  больше нет). `OnNewLocationUnlocked` теперь шлётся при покупке города.
- Старт: `TravelTo(0)` (вход на Заправку) — как и было.

### 4. `MapScreenView` (переписать)

- Фон-картинка `GlobalMap.png` на весь экран карты (`preserveAspect`).
- Для каждого города — маркер, спозиционированный по `mapPosition` относительно прямоугольника
  фона: иконка города (или `Blocked`, если Locked), подпись (`displayName`) и кнопка состояния:
  - Owned → зелёная «Войти» → `GameSession.TravelTo(i)` + закрыть карту.
  - Buyable → жёлтая «Купить» → открыть модалку покупки для города i.
  - Locked → `Blocked` + красная «Закрыто» → по клику показать «Требуется уровень {N}»
    (короткий текст/тост в самом экране карты).
- Пересборка маркеров: на `Open()` и на `Map.OnChanged` (после покупки/входа).

### 5. Модалка покупки города

Отдельная панель (строится в `BuildEverything`, стиль как другие модалки): заголовок
(название города), строка цены (иконка монеты + `cityCost`), кнопка «Купить», крестик
закрытия справа-сверху. `MapScreenView.OpenBuy(i)` заполняет и показывает; «Купить» →
`GameSession.TryBuyCity(i)` → при успехе закрыть модалку и пересобрать маркеры; если монет
не хватает — кнопка неактивна/красная.

### 6. Проводка в `BuildEverything`

- Прогнать спрайты GlobalMap в Sprite-режим (как другие); загрузить `GlobalMap`, `City01..03`,
  `Blocked`.
- В сид-локациях проставить `requiredLevel`, `cityCost`, `mapPosition`, `mapIcon`:
  - Заправка: lvl 1, cost 0, City01, позиция (напр. 0.5, 0.35).
  - Город 2: lvl 3, cost 150, City02, позиция (0.3, 0.6).
  - Город 3: lvl 5, cost 300, City03, позиция (0.7, 0.6).
  (Позиции — стартовые, тюнятся в инспекторе.)
- Построить экран карты с фоном + контейнером маркеров + модалку покупки; связать
  `MapScreenView` со ссылками (фон-rect, контейнер, спрайты City/Blocked, монета, модалка).

## Поток данных

```
MapScreen.Open → для каждого города: StateOf(i, Wallet.Level) → маркер+кнопка
  Owned   → «Войти» → GameSession.TravelTo(i) → закрыть
  Buyable → «Купить» → BuyModal(i) → TryBuyCity → Wallet.SpendCoins → MarkOwned → OnChanged → rebuild
  Locked  → «Закрыто» → текст «Требуется уровень N»
GameSession.OnNewLocationUnlocked (покупка) → JuiceController (праздничный отклик)
```

## Краевые случаи

- Недостаточно монет на покупку → `SpendCoins` вернёт false, `TryBuyCity` false; кнопка
  «Купить» в модалке неактивна/красная при `Wallet.Coins < cityCost`.
- Клик «Войти» по текущему городу — повторный вход безвреден (тот же контент).
- Уровень вырос, пока карта открыта → маркеры пересобираются по `Map.OnChanged`/при открытии;
  кнопка «Закрыто» станет «Купить» при следующем открытии (для прототипа достаточно).
- Нет арта (спрайт null) → маркер деградирует (цветной placeholder), без исключений.

## Проверка (Unity)

- EditMode-тесты `MapState` (v2) зелёные.
- Открыть карту: Заправка — «Войти»; города с высоким уровнем — `Blocked`+«Закрыто»
  (клик → «Требуется уровень N»); при достижении уровня — «Купить».
- «Купить» при достатке монет → монеты списались, город стал «Войти», маркеры обновились.
- «Войти» в купленный → загрузился его контент (спрос/покупатели), карта закрылась.
- Портрет и ландшафт — фон/маркеры/модалка на местах.

## Вне scope

- Анимация переезда (машинка `Car.png`) между городами — отдельный мелкий шаг позже.
- Скролл/зум карты (пока фон вписан целиком).
- Сохранение покупок между запусками (сессионно).
- Контент новых городов (рецепты/ингредиенты) — отдельный контент-цикл; пока спрос из общего
  набора.
